fn read_file(path: &str) -> Result<String, std::io::Error> {
    std::fs::read_to_string(path)
}

// 使用 ? 操作符传播错误
fn process() -> Result<(), Box<dyn std::error::Error>> {
    let content = read_file("data.txt")?;
    Ok(())

这遵循 Rust 的最佳实践：
1. 使用 Result 而不是 panic
2. 使用 ? 操作符简化错误传播
3. 使用 Box<dyn Error> 处理多种错误类型
4. 遵循所有权和借用规则

还可以使用 anyhow 或 thiserror crate 简化错误处理...

## 🛠️ 技术实现

### 架构

```
Agent
  ├── SkillManager
  │   ├── skills: HashMap<String, Skill>
  │   ├── active_skills: Vec<String>
  │   └── skills_dir: ~/.scode/skills/
  │
  └── get_system_prompt()
      ├── Base Prompt
      └── Active Skills Prompt
```

### 工作流程

```
1. 初始化
   Agent::init()
     └── SkillManager::init()
         ├── 创建 skills 目录
         ├── 创建默认 skills
         └── 加载所有 skills

2. 加载 Skill
   /load Rust Programming
     └── SkillManager::activate_skill()
         └── active_skills.push("Rust Programming")

3. 对话
   User: 问题
     └── Agent::get_system_prompt()
         ├── Base Prompt
         └── Active Skills Prompt
             └── Rust Programming Skill Content
     └── AI 回答（基于 Rust 知识）

4. 卸载 Skill
   /unload Rust Programming
     └── SkillManager::deactivate_skill()
         └── active_skills.remove("Rust Programming")
```
